ФИДО IP-комплект fidoip может работать в двух режимах:

I) Как клиент(для поинтов) - для подключения к другому узлу по сети Интернет;
II) Как сервер(для системных операторов узлов) - для предоставления доступа к сети FIDONet клиентам(вашим поинтам) и доступа другим узлам Фидо.

-----------------------------------------------------------
I) Если вы - поинт и хотите подключиться к узлу(вариант I):
-----------------------------------------------------------
 1) Разархивируйте файл fidoip_win-1.0.5_4.zip в корневую директорию любого диска.
 Для того чтобы комплект работал, он должен находиться в корне любого диска в 2-х директориях - usr и home!
 (например - d:\usr и d:\home, c:\usr и c:\home и т.д.);

 2) Запустите файл \usr\setup_config.vbs, предварительно отредактировав в нём восемь строк:

 const yourFullName      = "Vasiliy Pampasov"        	// Тут в кавычках следует указать Ваше имя и фамилию на английском; 
 const yourFTNAddress    = "2:5020/828.555"        	// Тут в кавычках следует указать Ваш FTN-адрес, который соощит Вам Ваш босс; 
 const yourStationName   = "MyStation"                   	// Тут в кавычках следует указать название Вашей станции, можете выбрать любое слово; 
 const yourLocation      = "Moscow, Russia"            	// Тут в кавычках следует указать Ваш город и страну; 
 const yourUplinkName    = "Kirill Temnenkov       	// Тут в кавычках следует указать имя Вашего босса; 
 const uplinkFTNAddress  = "2:5020/828"              	// Тут в кавычках следует указать FTN адрес узла Вашего босса; 
 const uplinkServerName  = "temnenkov.dyndns.org"	// Тут в кавычках следует указать DNS-адрес узла Вашего босса(hostname); 
 const yourPassword      = "12345678"     		// Тут в кавычках следует указать пароль на сессию, который Вам сообщит Ваш босс.

 3) Зайдите в директорию \usr\bin и щелкните мышкой на файле rs.bat или ярлыке со значком с изображением собачей будки (rs.bat) на Рабочем столе Windows;

 4) После того как вы соединитесь и получите сообщения, запустите редактор GoldEd-NSF щелкнув мышкой на файле \usr\bin\ge.bat или ярлыке с логотипом фидошной собачки (ge.bat) на Рабочем столе. Написав письма, закройте редактор, запустите снова скрипт rs.bat для отправки ваших сообщений. 

 5) Это всё! Для более подробного описания работы комплекта см. файл fido.rus.win.txt каталоге \usr\doc,
либо в корневом каталоге архива fidoip*.tar, либо на сайте http://sourceforge.net/apps/mediawiki/fidoip.

------------------------------------------------------------------------------------
II) Если ты - системный оператор узла FIDONet(босс) или хочешь им стать(вариант II):
------------------------------------------------------------------------------------
 1) Разархивируй файл fidoip_win-1.0.5_4.zip в корневую директорию любого диска.
 Для того чтобы комплект работал, он должен находиться в корне любого диска в 2-х директориях - usr и home!
 (например - d:\usr и d:\home, c:\usr и c:\home и т.д.);

 2) Запусти файл \usr\setup_node.vbs, предварительно отредактировав в нём десять строк:

 const yourFullName         = "Vasiliy Pampasov"  // Тут в кавычках следует указать твое имя и фамилию на английском; 
 const yourNodeAddress      = "2:5020/788"        // Тут в кавычках следует указать FTN-адрес вашего узла, который сообщит тебе твой региональный координатор; 
 const yourStationName      = "MyStation"         // Тут в кавычках следует указать название твоей станции, можете выбрать любое слово; 
 const yourLocation         = "Moscow, Russia"    // Тут в кавычках следует указать твой город и страну;
 const yourDNSNAMEorIP      = "192.168.4.7"       // Тут в кавычках следует указать DNS имя твоего узла или IP-адрес;
 const your1stPointPassword = "12345678"          // Тут в кавычках следует указать твой пароль(не длиннее 8-ми символов) для поинта .1, для доступа и чтения конференций с другого компьютера(по сети);
 
 const yourUplinkName     = "Kirill Temnenkov"       	// Тут в кавычках следует указать имя аплинка, с которым ты предварительно договорился о соединении; 
 const uplinkFTNAddress   = "2:5020/828"              	// Тут в кавычках следует указать FTN адрес узла аплинка; 
 const uplinkServerName   = "temnenkov.dyndns.org"	// Тут в кавычках следует указать DNS-адрес узла аплинка(hostname); 
 const yourUplinkPassword = "09876543"     		// Тут в кавычках следует указать пароль на сессию, который Вам сообщил твой аплинк.

 После того, как ты щелкнул два раза на файле setup_node.vbs конфигурация узла уже полностью готова к работе.  
 В появившемся окне редактора GoldED прочти письмо от разработчика с дальнейшими инструкциями по работе с 
 "Утилитами администрирования узла fidoip".

 3) Зайди в директорию \usr\bin и щелкни мышкой на файле rs.bat - для проверки связи с аплинком или щелкни на ярлыке "Polling-test" на Рабочем столе Windows. 
 
 4)  Затем нужно запустить сам сервис binkd и встроенный в fidoip планировик заданий fxcron - чтобы твои поинты и даун-линки могли подключиться и забрать почту и сообщения эхоконференци, а твой узел автоматически опрашивал другие 
линки по графику.

При первоначальной установке узла - ничего запускать не нужно, конфигуратор setup_node.vbs все запустит самостоятельно.
После перезагрузки Windows для запуска файла \usr\bin\binkdsrv.bat на Рабочем столе Windows есть ярлык. 
Просто щелкни на нем и сервисы binkd и fxcron запустятся.

В открывшемся окне видно, что происходит по время соединений. 
При сворачивании, окно с сервисом binkd прячется в системную Панель Задач
(трей), для его разворачивания щелкни в правом углу панели задач на значке.

Для тестирования сервиса binkd ты можешь установить fidoip на другом компьютере для первого системного поинта MYNODE-ADDRESS.1 , настроив его при помощи конфигуратора для поинтов setup_config.vbs (для Windows) или 
setup_config.bash (для Linux, FreeBSD и DragonFly BSD - введи свое имя, номер поинта .1 и пароль, которые ты выбрал при первоначальной установке.

Протестируй узел и убедись, что все работает нормально. 

В случае возникновения проблем смотри лог сервиса binkd в файле 
\home\fido\binkd.log, лог планировщика fxcron \home\fido\crond.log,
лог тоссера \home\fido\hpt.log.

 5) Затем нужно добавить ярлык \usr\bin\binkdsrv.bat  в меню Автозагрузки (_Пуск>Программы>Автозагрузка_), чтобы сервис запускался автоматически при перезагрузке. Чтобы сервис стал доступен снаружи, на твоём маршрутизаторе(firewall'е) нужно открыть доступ к этому сервису(TCP порт 24554), чтобы он был доступен из  Интернета. 

 6) Для того, чтобы периодические задания, выполняющееся каждый день ночью и ежемесячно(\usr\bin\fido.dayly.bat и \usr\bin\fido.monthly.bat), публикующее в локальные эхи статистику и правила, работали правильно, нужно поставить Perl для Windows. Мною опробованы две сборки Perl для Windows - ActiveState Perl(http://www.activestate.com) и Strawberry Perl(http://strawberryperl.com/). Обе версии работали со скриптами правильно. Ниже приведена краткая инструкция по установке ActiveState Perl, но для Strawberry Perl она похожая. Нужно загрузить с сайта:

http://www.activestate.com/activeperl/downloads

один из двух исполняемых файлов ActiveState Perl и установить его. На странице загрузки присутствуют 2 версии ActiveState Perl: 
для 32-битной Windows(x86) и для Windows 64-bit(x64). 

Скачай одну из версий для твоей Windows и установи на компьютер, щелкнув на скачанном файле ActivePerl-version-platform-buildversion.msi.
 
 7) По-умолчанию запуск переодических заданий осуществляется при помощи встроенного в fidoip планировщика заданий fxcron.

Сами периодические задания планировщика задач fxcron заданы в текстовом файле \usr\fidoconf\crontab.cfg

По-умолчанию их четыре - первые два публикуют правила, статистику, чистят старые файлы и т.д, третье (rs.bat) - выполняет поллинг - обмен сообщениями с другими узлами каждые 22 минуты), четвертое (toss.bat) выполняет тоссинг каждые 7 минут для ускорения обмена сообщениями для поинтов твоего узла.

Значения подобраны оптимальным образом, но если тебе нужно что-то изменить, к примеру, увеличить интервал времени запуска или уменьшить или добавить еще какое-то задание - просто отредактируй файл \usr\fidoconf\crontab.cfg.

Формат файла - \usr\fidoconf\crontab.cfg - стандартный crontab:

мин  часы   день-месяца   месяц   день-недели  команда

Поле         Допустимые значения
-----         --------------
минуты        0-59
часы          0-23
день-месяца   1-31
месяц         1-12 (или имена: jan, feb, mar, apr, may, jun, jul, aug, sep,
              oct, nov, dec)
день-недели   0-7 (0 или 7 это Воскресенье, или по именам: Sun, Mon, Tue, 
              Wed, Thu, Fri, Sat)     

Символы  */  - значит выполнять через определенный период времени

*/22      *      *      *      *      @D:/usr/bin/rs.bat

Означает - запускать поллинг каждые 22 минуты каждый день. 

Отредактировав файл \usr\fidoconf\crontab.cfg, перезапусти планировщик fxcron щелкнув на значке (пиктограмме часов) и выбрав в меню "Reload crontab file" или щелкнув на файле \usr\bin\crontask.vbs 

 8) для управления узлом fidoip, для добавления/удаления новых линков и поинтов используй "Утилиты администрирования узла fidoip", которые находятся в каталоге \usr\tools. В этих утилитах реализована защита от неправильного ввода, которые можно допустить при прямом редактировании конфигурационных файлов вручную. Более подробно об использовании этих утилит, см. документацию, а также первое письмо от разработчика в редакторе GoldED;

 9) Это всё! Для более подробного описания работы узлового комплекта см. файл fido.rus.win.txt в каталоге \usr\doc, либо на сайте http://sourceforge.net/apps/mediawiki/fidoip
 Для информации о параметрах конфигурационных файлов мейлера binkd, тоссера hpt и редактора GoldED, см. документацию каталоге node/doc архива fidoip*.tar.
