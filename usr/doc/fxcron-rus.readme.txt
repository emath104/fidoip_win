
fxcron - простой и легкий планировщик заданий для Windows (W2k - W7)


Эта программа - планировщик заданий, программный код которой частично заимствован из
популярного в Unix/Linux планировщика заданий crond. После запуска fxcron, его значок(часы)
появляется в правом углу Панели Задач Windows(в трее). 

fxcron не работает как Служба Windows(Сервис). Использование планировщика и его обслуживание 
намеренно сделаны как можно проще. Планировщик написан на чистом C( быстр, компактен, занимает в памяти
около 2-ух мегабайт) и совместим с форматом файлов периодических заданий crontab. 

Все периодические задания задаются в файле crontab. Планировщик может запускать программы и скрипты как
в интерактивно, так и в фоне - т.е. незаметно для пользователей, не пугая их внезапно появляющимися окнами.
 
Формат файла crontab для fxcron

В основном файл crontab file для fxcron совместим в форматом crontab демона crond Unix/Linux.

мин  часы   день-месяца   месяц   день-недели  команда

Каждое задание располагается на отдельной строке, параметры между значениями отделются одним или несколькими
пробелами. Пустые строки с пробелами игнорируются.

Поле         Допустимые значения
-----         --------------
минуты        0-59
часы          0-23
день-месяца   1-31
месяц         1-12 (или имена: jan, feb, mar, apr, may, jun, jul, aug, sep,
              oct, nov, dec)
день-недели   0- 7 (0 или 7 это Воскресенье, или по именам: Sun, Mon, Tue, 
              Wed, Thu, Fri, Sat)     

Имена и названия месяцев и дней не зависят от регистра, могут быть строчные и прописные.
Каждое поле может содержать одно значение, список разделенный запятой (,) или диапазон.
Диапазон между двумя значениями задается знаком тире (-), к примеру mon-fri.
(*/)  - значит выполнять через определенный период времени, к примеру, */10 означает
выполнять задание каждые 10 минут. 

Комментарии начинаются с символа (#), все дальнейшие символы после этого знака правее игнорируются 
 
### пример файла crontab  #############################################################

# Запуск резервного копирования каждый рабочий день в 12:15 (в фоне)

15      12      *       *     mon-fri   @c:/scripts/daily-backup.cmd

# Еженедельное резервное копирование в 4 часа дня в пятницу (интерактивно)

0       16      *       *     fri       c:/scripts/weekly-backup.cmd

# Ежемесячное резервное копирование в 1 час ночи первого числа, если компьютер
# выключен, повторить задание после включения

0        1      1       *      *  !c:/scripts/montly-backup.cmd

# Запускать rsync каждые два дня, в 1:00 (в фоне)

0        1    */2       *     *   @c:/scripts/rsync.cmd

### пример файла crontab #############################################################

Более подробно описание формата crontab см. также
http://ru.wikipedia.org/wiki/Crontab

Использование:
fxcron [ОПЦИИ]
-f --file  <файл crontab>       задает файл crontab
-l --logfile <logfile>          задает log файл для журналирования событий
-v --verbose                    увеличивает информативность вывода в log файл/ уровень журналирования по-умолчанию ПРЕДУПРЕЖДАТЬ(WARN)
-q --quiet                      уменьшает информативность вывода / уровень журналирования по-умолчанию ПРЕДУПРЕЖДАТЬ(WARN)

Пример:
fxcron -f c:/scripts/crontab.txt -l c:/scripts/fxcronlog.txt -vv
запускать fxcron с уровнем журналирования ИНФОРМИРОВАТЬ(INFO)

После редактирования файла crontab для того, чтобы планировщик fxcron перечитал задания, щелкните на значке в трее и выбрите в меню "Reload crontab file".

Также возможно запустить fxcron без опций и задать файл crontab файл журнала в графическом интерфейсе,
щелкнув мышкой на значке fxcron.

Также из графическом интерфейсе возможно выбрать сохранение состояния заданий в реестре Windows. 
По-умолчанию fxcron не изменяет реестр Windows, если это не нужно пользователю. Но если эта опция выбрана,
после каждого запуска задания в реестр записывается временная метка последнего выполнения задания.
В случае перезагрузки или выключения компьютера эта метка позволяет fxcron запускать после включения
компьютера пропущенные задания.
